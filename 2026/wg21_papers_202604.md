モジュール内での`#depend`で宣言された依存関係をエクスポートできるようにする提案。

`#depend`は`std::embed`でインクルードできるリソースを宣言するためのプリプロセッシングディレクティブです。`std::embed`では読み取ったリソースをパースしてそこから次の読み込みリソースのパスを取得して再び`std::embed`するといったことが可能になりますが、この時にC++コードのコンパイル時にリソース探索を行わないようにするために、あらかじめ読み込めるリソースの範囲を制限（あるいは宣言）しておくのが`#depend`です。

`std::embed`は`#depend`で宣言されたリソースしか読み込むことができません。

`std::embed`/`#depend`に関しては以前の記事を参照

- [P1040R7 `std::embed` and `#depend` - WG21月次提案文書を眺める（2024年12月）](https://onihusube.hatenablog.com/entry/2025/05/18/195724#P1040R7-stdembed-and-depend)

`#depend`をモジュールで記述した際にその依存関係情報をどのように扱うか（モジュール外部への影響をどうするか）には、`#include`同様にモジュール内で閉じたものとして扱うか、そのモジュールをインポートした側でも有効にするかの2つの選択肢があります。

`#include`と同様に扱う以外の選択肢があり得るのは、`#depend`が宣言するのが外部依存関係（C++ソースコードではない依存関係）だからです。内部依存関係は`#include`を使わなくてもモジュールの再`export`によってモジュールから依存関係を公開するかを制御できますが、外部依存関係にはそのような方法がありません。したがって、`#include`と同様に扱うと不便が生じる可能性があります。

P1040の議論の中でこの問題は認識され、現在は`#include`と同じくモジュールからその依存関係の宣言は漏洩しないようになっています。

仮にモジュール内での`#depend`宣言がそれをインポートする側にも伝播する場合、モジュール内でプライベート部分が容易に外部に公開されてしまう事態が発生します。

```cpp
export module deps;

#depend <sdk/private/​**>       // モジュールプライベートな依存関係
#depend <sdk/everything/​**>    // 
// #depend宣言がこのモジュールをインポートした側にも伝播する場合
```
```cpp
import deps;

int main () {
  std::embed("sdk/everything/meow.wav"); // ok
  std::embed("sdk/private/super_secret_sauce.mix"); // uh-oh, ok?!
  return 0;
}
```

このようなことが起こらないようにするために一律に漏洩しないようにされています。しかし、この例の`sdk/everything/​**`のようにむしろモジュールから公開したい外部リソースも想像することができます

この提案はこのような問題を解決するために、`#depend`による外部依存関係の宣言をモジュールから`export`するかどうかを選択できるようにするものです。これは`#depend`の直後に`export`を付加したプリプロセッシングディレクティブによって外部依存関係を宣言することで制御します。

```cpp
/// deps.c++m
export module deps;

#depend <sdk/private/​**>           // モジュール（翻訳単位）で内部的
#depend export <sdk/everything/​**> // モジュールから公開する
```
```cpp
/// main.cpp
import deps;

int main () {
  std::embed("sdk/everything/meow.wav");            // ok
  std::embed("sdk/private/super_secret_sauce.mix"); // ng
  return 0;
}
```

`#depend export`で宣言された外部依存関係はそのモジュールからエクスポートされ、インポートした側でも`#depend`されているかのように扱うことができます。一方、単なる`#depend`による外部依存関係の宣言はモジュール（翻訳単位）に閉じたものになり、インポートした側からは利用できません。

この提案では今のところ、`#depend export`によって宣言された依存関係はそのモジュールから公開されるもので、`#depend`による依存関係はその翻訳単位に閉じたものです。同じモジュール内で共有するがモジュール外には公開しない（モジュールリンケージの様に）という制御方法は提供していません。
